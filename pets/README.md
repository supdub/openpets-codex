# Pet catalog

Every directory containing `pet.json` is a catalog entry. The two runtime files are the manifest
and referenced spritesheet; repository metadata, previews, source notes, and asset terms stay here
for contributors and are not installed into Codex.

Use `openpets list` to browse, `openpets preview <id>` to inspect, and `openpets install <id>` to
install. New contributors should start with [Creating a pet](../docs/CREATE_A_PET.md).
